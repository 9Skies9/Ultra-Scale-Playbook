## Scatter

Scatter: one source starts with the full tensor and splits it into pieces, sending one piece to each GPU.

Example:

-   source has \[A | B | C | D\]
    
-   after scatter:
	-   GPU 0 gets A
	-   GPU 1 gets B
	-   GPU 2 gets C	    
	-   GPU 3 gets D


## Gather

Gather: the reverse shape-wise. Each GPU starts with one piece, and one destination collects all pieces into the full tensor.

Example:

-   GPU 0 has A
-   GPU 1 has B
-   GPU 2 has C
-   GPU 3 has D

-   after gather, one chosen GPU has \[A | B | C | D\]


## Broadcast

Broadcast: one source has a tensor, and it sends the exact same full copy to everyone.

Example:

-   GPU 0 has X
-   after broadcast:
	-   GPU 0 has X
	-   GPU 1 has X
	-   GPU 2 has X
	-   GPU 3 has X


## Reduce

Reduce: all GPUs start with same-shaped tensors, and they are combined by some operation, usually sum, onto one destination GPU.

Example:

-   GPU 0 has X0
-   GPU 1 has X1
-   GPU 2 has X2
-   GPU 3 has X3

-   after reduce-to-GPU-0:
	-   GPU 0 has X0 + X1 + X2 + X3


## All Reduce

All-reduce: same as reduce, except everybody gets the final reduced result.

Example:
-   each GPU starts with its own tensor
-   after all-reduce:
	-   every GPU has X0 + X1 + X2 + X3

This is the classic data-parallel gradient averaging pattern.


## All Gather

All-gather: each GPU starts with a shard, and every GPU receives all shards and reconstructs the full tensor.

Example:
-   GPU 0 has A
-   GPU 1 has B
-   GPU 2 has C
-   GPU 3 has D

-   after all-gather:
	-   every GPU has \[A | B | C | D\]


## Reduce Scatter

Reduce-scatter: each GPU starts with a full-shaped partial contribution, those are reduced together, and then the final reduced tensor is split into shards, one shard per GPU.

Example:

-   GPU 0 has partial tensor P0
-   GPU 1 has P1
-   GPU 2 has P2
-   GPU 3 has P3

First reduce:

P = P0 + P1 + P2 + P3

Then scatter:

-   GPU 0 gets first shard of P
-   GPU 1 gets second shard of P
-   GPU 2 gets third shard of P
-   GPU 3 gets fourth shard of P

This is very common when going from tensor-parallel output back into sharded form.



## Conclusion

The clean mental model is:

- Broadcast = one full copy becomes many full copies
- Scatter = one full tensor becomes many pieces
- Gather = many pieces become one full tensor
- All-gather = many pieces become many full copies
- Reduce = many tensors combine into one full result on one GPU
- All-reduce = many tensors combine into one full result on every GPU
- Reduce-scatter = many tensors combine, then that result is split into pieces

A good way to remember the names:
- “all” means every GPU receives the result.
- “gather” means assemble pieces.
- “scatter” means distribute pieces.
- “reduce” means combine with an operation, usually sum.

So:
-   all-gather = assemble pieces everywhere
-   reduce-scatter = combine everything, then distribute pieces

There are also a few related terms people use around this:
- Sharding: splitting a tensor or model state across GPUs. Not itself a collective, just the storage/layout idea.
- Replica / replicated: every GPU holds the same full copy.
- Partitioning: generic word for dividing data/model across devices.
- Point-to-point communication: direct send/recv between specific GPUs, not a full collective over everybody.
- Barrier / synchronization: everyone waits until all processes reach the same point.


The practically relevant ones for LLM training are mostly these four:
-   all-reduce
-   all-gather
-   reduce-scatter
-   broadcast

And the most important pair to keep straight is:
- all-gather: shards -> full copies on all GPUs
- reduce-scatter: partial full contributions -> final shards on each GPU

That pair shows up constantly in tensor parallelism and sequence/context parallelism.
